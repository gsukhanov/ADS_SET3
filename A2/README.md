# A2. Анализ MERGE + INSERTION SORT.

В папке содержится файл `A2.cpp`. Скомпилировав его и запустив с опциями resultx, breakpoint можно получить `.csv` файл с названием `resultx.csv`, в котором находятся результаты работы `MERGE SORT + INSERTION SORT` с точкой применения `INSERTION SORT` равной breakpoint. Для получения чистого `MERGE SORT` значение breakpoint необходимо выставить равным 1.
Все данные собирались с использованием ключа компиляции `-O3`.

Собрав все данные в одну таблицу `A2.xlsx` были составлены 4 графика:

1. Время сортировки... - содержат сравнение скоростей сортировки различных массивов с различными точками применения `INSERTION SORT`. 
2. Сортировка Merge + Insertion. Здесь сравнивается поведение наиболее эффективной согласно графикам вариации алгоритма на различных массивах.

Из данных можно сделать следующие выводы:

1) Наибольшую эффективность показывают небольшие значения breakpoint, причем 15 и 50 - приблизительно равную.
2) Сортировка имеет более высокую эффективность если размер массива имеет вид 2^n * breakpoint, что заметно по скачкам для высоких breakpoint, причем чем выше breakpoint, тем заметнее такой скачок.
3) Несмотря на то, что значение breakpoint равное 150 не столь эффективно, оно все еще зачастую эффективнее чистого `MERGE SORT`.
4) На удивление, сортировка эффективнее для развернутого массива, чем для случайного, но всё же менее эффективна, чем для почти отсортированного.
